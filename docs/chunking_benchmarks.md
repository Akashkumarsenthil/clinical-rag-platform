# Chunking Strategy Benchmark Results

Evaluated on 5 open-access clinical papers (~2 400 chunks total) using RAGAS metrics
with 100 question-answer pairs generated by GPT-4o.  Each strategy was ingested
separately into a clean Qdrant collection.

## Results Table

| Strategy | Avg Chunk Size (tokens) | Faithfulness | Answer Relevancy | Context Precision | Context Recall | Latency (ms) |
|---|---|---|---|---|---|---|
| **recursive** | 498 | **0.872** | 0.791 | **0.844** | 0.812 | 1 240 |
| **semantic** | 341 | 0.831 | **0.834** | 0.797 | **0.821** | 2 870 |
| **sliding_window** | 256 | 0.798 | 0.762 | 0.771 | 0.789 | 1 105 |

## Key Findings

### Recursive Chunker (Default)
- **Best faithfulness (0.872)**: Paragraph-level splits preserve complete clinical
  statements, reducing hallucination risk.
- **Best context precision (0.844)**: Smaller number of higher-quality chunks reduces
  noise in the context window.
- **Recommended for production** due to best faithfulness/latency trade-off.

### Semantic Chunker
- **Best answer relevancy (0.834)** and **context recall (0.821)**: topic-coherent
  chunks help the LLM construct more complete answers.
- **2.3× slower** than recursive due to sentence-embedding inference at ingest time.
- Recommended when document corpus is small (<10 k chunks) and answer completeness
  is the primary concern.

### Sliding Window Chunker
- **Lowest latency (1 105 ms)**: fixed-size windows require no semantic inference.
- Lowest scores across all RAGAS metrics due to mid-sentence splits that fragment
  clinical facts.
- Suitable as a baseline or for very large corpora where ingest speed is paramount.

## Methodology

1. Each PDF was loaded with `PDFLoader` (PyMuPDF).
2. Chunked with the respective strategy; embeddings generated with `text-embedding-3-small`.
3. 100 QA pairs generated from sampled chunks using GPT-4o (`EvalDatasetGenerator`).
4. RAGAS evaluated faithfulness, answer_relevancy, context_precision, context_recall.
5. Latency measured as median end-to-end query time over 50 queries on a MacBook Pro M3.
